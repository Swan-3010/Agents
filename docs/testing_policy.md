# Testing Policy

**Проект:** Yandex Mail Agent  
**Версия:** 0.1.0  
**Статус:** Active

---

## Обязательное правило: test-first

До написания любой новой функции обязательно:

1. Описать test cases по категориям `unit` / `integration` / `contract` / `e2e`
2. Сохранить документацию в `docs/` (используя шаблоны)
3. Создать stub-тесты в `tests/`
4. Только потом писать имплементацию

---

## Категории тестов

| Категория | Цель | Папка | Требует окружения |
|---|---|---|---|
| `unit` | Проверка логики отдельных компонентов | `tests/unit/` | Нет |
| `integration` | Проверка взаимодействия IMAP/SMTP | `tests/integration/` | IMAP креденциалы |
| `contract` | Обратная совместимость схем | `tests/contract/` | Нет |
| `e2e` | Полный цикл агента | `tests/e2e/` | Полное окружение |

---

## Нумерация тест-кейсов

```
unit/TC-0XX
integration/TC-1XX
contract/TC-2XX
e2e/TC-3XX
```

---

## Stub-соглашение

- Stub-тесты используют `pytest.skip()` с пояснением
- Имплементация заполняется после написания кода
- PR не мержится без выполнения `pytest -v` без ошибок

---

## Инструменты

- **pytest** — основной фреймворк
- **pytest-asyncio** — async тесты
- **pytest-playwright** — e2e браузерные тесты
- **mypy** — статическая типизация
- **ruff** — линтинг
